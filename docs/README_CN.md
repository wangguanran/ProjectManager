## 开始使用

### 🚀 新用户快速路径
1. **阅读快速开始**: 从[快速开始指南](user-guide/getting-started_CN.md)开始
2. **学习基本命令**: 查看[命令参考](user-guide/command-reference_CN.md)
3. **配置项目**: 学习[配置管理](user-guide/configuration_CN.md)
4. **实践操作**: 按照[用户指南](user-guide/README_CN.md)的学习路径

### 👨‍💻 开发者路径
1. **了解架构**: 查看[系统架构](development/architecture_CN.md)
2. **设置环境**: 按照[开发指南](development/README_CN.md)设置开发环境
3. **编写测试**: 学习[测试策略](development/testing_CN.md)
4. **贡献代码**: 参考[贡献指南](development/README_CN.md)

### 🚀 高级用户路径
1. **深入功能**: 查看[功能文档](features/)
2. **性能优化**: 学习[脚本和自动化](development/scripts_CN.md)
3. **部署发布**: 参考[部署指南](deployment/)

## 文档标准

- **文件命名**: 使用小写字母和连字符（例如 `feature-name.md`）
- **语言**: 所有文档使用中文
- **格式**: Markdown格式，具有清晰的标题和结构
- **示例**: 包含实际示例和用例
- **链接**: 在文档内使用相对链接
- **测试用例**: 需求文档包含对应的自动化测试用例

## 为文档做贡献

添加新功能或进行更改时：

1. **更新相关文档**在适当的类别中
2. **使用清晰、简洁的语言**并提供示例
3. **遵循现有结构**和命名约定
4. **包含代码示例**在适当的地方
5. **测试文档**以确保准确性
6. **添加测试用例**到需求文档中

## 支持

如果您发现文档问题或需要澄清：

1. 检查相关部分的现有信息
2. 在文档中搜索类似主题
3. 在GitHub上提出具体问题
4. 通过拉取请求贡献改进
